Report errors to [support@konstantly.com](mailto:support@konstantly.com)

Base URL: `https://YOURSITE.konstant.ly/openapi/v1`

# Courses

## List Courses

<RequestExample>
```bash
curl --request GET \
  --url 'https://YOURSITE.konstant.ly/openapi/v1/courses' \
  --header 'X-API-KEY: YOUR_API_KEY'
```
</RequestExample>

```json GET /courses
{
  "parameters": {
    "query": {
      "offset": {
        "type": "integer",
        "description": "Entry number to start listing from",
        "required": false
      }
    }
  }
}
```

Returns a list of courses (20 per page).

## Get Course

```json GET /courses/{courseId}
{
  "parameters": {
    "path": {
      "courseId": {
        "type": "integer",
        "description": "Course ID",
        "required": true
      }
    }
  }
}
```

Returns detailed information about a specific course.

## Assign Course

```json POST /courses/{courseId}/assignments
{
  "parameters": {
    "path": {
      "courseId": {
        "type": "integer",
        "description": "Course ID",
        "required": true
      }
    }
  },
  "body": {
    "users": {
      "type": "array",
      "items": {
        "type": "integer"
      },
      "description": "User IDs to be assigned"
    },
    "groups": {
      "type": "array",
      "items": {
        "type": "integer"
      },
      "description": "Group IDs to be assigned"
    },
    "assignAt": {
      "type": "integer",
      "description": "Timestamp when the course should be assigned (UTC)"
    },
    "deadlineAt": {
      "type": "integer",
      "description": "Deadline timestamp (UTC)"
    }
  }
}
```

# Users

## List Users

```json GET /users
{
  "parameters": {
    "query": {
      "query": {
        "type": "string",
        "description": "Filter by email",
        "required": false
      },
      "offset": {
        "type": "integer",
        "description": "Pagination offset",
        "required": false
      }
    }
  }
}
```

Returns a list of users (sorted by name, 20 entries per page).

## Create User

```json POST /users
{
  "body": {
    "type": "object",
    "properties": {
      "email": {
        "type": "string",
        "required": true
      },
      "firstName": {
        "type": "string",
        "required": true
      },
      "lastName": {
        "type": "string",
        "required": true
      }
    }
  }
}
```

Creates a new user in the system.

## Get User by Email

```json GET /users/byEmail/{userEmail}
{
  "parameters": {
    "path": {
      "userEmail": {
        "type": "string",
        "description": "User email",
        "required": true
      }
    }
  }
}
```

# Webhooks

The API provides several webhook endpoints for different events:

## Course Events
- `/Webhooks/Course/Published` - Course published
- `/Webhooks/Course/Unpublished` - Course unpublished
- `/Webhooks/Course/Assigned` - Course assigned
- `/Webhooks/Course/Started` - Course started
- `/Webhooks/Course/Progress` - Course progress
- `/Webhooks/Course/Finished` - Course finished

## User Events
- `/Webhooks/User/Created` - User created
- `/Webhooks/User/Edited` - User edited
- `/Webhooks/Group/User/Created` - User added to group

Each webhook receives a POST request with the following structure:

```json
{
  "occuredAt": "integer (timestamp)",
  "type": "string (webhook type)",
  "body": "object (event-specific data)"
}
```

# Error Responses

The API uses conventional HTTP response codes to indicate the success or failure of an API request:

- `200` - Success
- `400` - Bad Request
- `404` - Not Found
- `422` - Unprocessable Entity
